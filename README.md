# cyber-divination

一个用 Rust 编写的 CLI/TUI 数字起卦程序，基于 `ratatui + crossterm` 实现。

## 目标

- 用明确状态机驱动一次完整的数字起卦仪式
- 每次 `Enter` 只即时生成一个 `2` 或 `3`
- 按易经由下至上的规则生成六爻，再按上至下渲染最终卦象
- 在终端内给出本卦、之卦、爻图和变爻说明

## 运行

```bash
cargo run
```

退出按键：

- `Enter`：推进仪式
- `q` / `Esc`：退出

## 模块结构

- `src/main.rs`
  负责组装模块与启动程序
- `src/tui.rs`
  管理终端 raw mode 与 alternate screen
- `src/app.rs`
  应用状态机、事件循环、历史记录与流程推进
- `src/ui.rs`
  `ratatui` 渲染层，只关心展示
- `src/entropy.rs`
  即时取熵逻辑，混合纳秒时间与 OS 安全随机源
- `src/yijing.rs`
  六爻映射、卦名查询、变爻格式化

## 状态机

流程固定为：

1. `Welcome`
2. `Casting`
3. `Assembling`
4. `ReverseConfirm`
5. `Result`

`Casting` 阶段共接收 18 次数字，每 3 次组成一爻，合计 6 爻。

## 即时随机机制

程序不会预生成任何数字。每次用户按下 `Enter` 时，才会：

1. 获取当前系统纳秒时间戳
2. 调用 `getrandom` 读取 OS 底层安全随机字节
3. 将两者与当前掷数编号混合
4. 立刻得到本次的 `2` 或 `3`

这样既保留“按键瞬间”的时机因素，也避免 UI 延迟展示伪装成实时随机。

## 卦象映射规则

- `2 + 2 + 2 = 6` 老阴
- `2 + 2 + 3 = 7` 少阳
- `2 + 3 + 3 = 8` 少阴
- `3 + 3 + 3 = 9` 老阳

六爻以底爻为第一个生成值。

索引计算采用二进制思想：

- 阳爻记为 `1`
- 阴爻记为 `0`
- 下卦占低 3 位
- 上卦占高 3 位
- 最终索引范围为 `0..=63`

## 变卦计算

- `6` 老阴变为阳，变后按 `7` 处理
- `9` 老阳变为阴，变后按 `8` 处理
- `7/8` 为静爻，不变化

如果存在变爻，结果页会同时展示本卦与之卦。

## UI 快照测试

项目包含基于 `ratatui::backend::TestBackend` 的界面快照测试，用来锁定：

- 欢迎页的布局与文案
- 结果页是否正确展示本卦、之卦与变爻信息

## 维护建议

- 要调整仪式文案，优先改 `src/app.rs`
- 要调整配色和布局，改 `src/ui.rs`
- 要替换随机策略，改 `src/entropy.rs`
- 要补充卦辞、爻辞或变卦，扩展 `src/yijing.rs`

更详细的维护说明见 [docs/architecture.md](docs/architecture.md)。
